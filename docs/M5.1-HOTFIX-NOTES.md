# ComfyUI Manager v0.5.1 Hotfix Notes

**Release date**: 2026-07-13
**Type**: hotfix over v0.5.0

## Bug fix

### WPF GUI 启动秒退 — Python service 无法启动

**症状:** 双击 `ComfyUI Manager.exe` 后,WPF 进程出现在任务栏约 6 秒后消失,没有 GUI 窗口弹出,Python service 未拉起,端口 7800 没有响应。

**Root cause:** `src/comfy_mgr/cli.py` 是 Typer app,但**缺少 `__main__.py`**。WPF 的 `PythonLauncher.cs:44-46` 用 `python -m comfy_mgr.cli serve` 启动 Python service 作为隐藏子进程;没有 `__main__.py` 时这个命令 silent-fail,Python 进程立刻退出。WPF 等 10s `/healthz` 超时后抛 `ServiceLaunchException` 退出。

**修复:** 新增 `src/comfy_mgr/__main__.py`(4 行 shim,re-export Typer `app`):

```python
from comfy_mgr.cli import app

if __name__ == "__main__":
    app()
```

**验证:** `python -m comfy_mgr.cli serve --port 7821` 启动 uvicorn,`/healthz` 返回 `{"status":"ok"}` HTTP 200,WPF → service 链路打通。
