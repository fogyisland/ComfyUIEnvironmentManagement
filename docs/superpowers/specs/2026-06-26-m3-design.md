# M3 节点生命周期深化 — 设计 Spec

**里程碑：** M3（在 M0 CLI 基础 + M1 GUI + M2 节点管理基础上）
**日期：** 2026-06-26
**状态：** 待用户审阅
**Base SHA：** e77d7ae（v0.2.0 tag）

---

## 0. 摘要

M3 在 M2 的节点扫描/启用/禁用基础上，深化节点生命周期管理：

1. **版本管理** — 升级 / 回滚 / 锁定（基于 git）
2. **依赖自动解析** — 扫描 requirements.txt + pyproject.toml + 冲突检测
3. **在线节点目录浏览** — 拉 ComfyUI Manager 官方 API，缓存到本地
4. **依赖冲突查询** — 调外部 API（M3 留 hook，未激活）

配套：本地 mock-free 架构（不内置 mock 后端服务），3 个新表 + 4 个 service + 1 个 install service + 2 个 HTTP client，自带 git portable 不依赖系统 git。

---

## 1. 设计目标 & 非目标

### 1.1 目标（M3 完成时）

- 用户能一键升级 / 回滚 / 锁定任意 git 来源节点
- 用户能查看每个节点的依赖列表，识别同 env 内依赖冲突
- 用户能浏览 ComfyUI Manager 官方节点目录，搜索并安装
- HTTP 不可达时优雅降级（缓存 + 离线提示）
- 不依赖用户机器装 git（项目自带 portable 版本）

### 1.2 非目标（明确不做）

- ❌ 定时自动升级 / 后台 watcher
- ❌ Semver 智能比较（用 commit SHA 精确回滚）
- ❌ 依赖图可视化（文字列表够用）
- ❌ AST_DYNAMIC 兜底（spread 解析仍 warning，留 M4）
- ❌ 安装进度条（用 BusyIndicator 简单标记）
- ❌ Web UI / SSH 远程管理（M0 spec §9.5 的横向扩展）
- ❌ Wiki 编辑界面（用户在 brainstorming 阶段撤回）
- ❌ 内置 mock 后端服务（用户撤回）
- ❌ 多用户 / 鉴权

### 1.3 暂留 hook（M4+ 激活）

- 🪝 `CompatHTTPClient` — base_url 可配但 M3 不激活
- 🪝 `catalog_cache_ttl` settings 字段 — M3 默认 3600s，UI 不暴露
- 🪝 `scanned_nodes.locked` 字段 — M3 在 scanned_nodes 加新列（详见 §3.5）

### 2.1 核心模块边界

| 模块 | 职责 | 依赖 |
|---|---|---|
| `VersionService` | git 升级/回滚/锁定 + version_history | git portable, VersionRepo, EventBus |
| `DepService` | 解析 requirements.txt/pyproject + 冲突检测 | DepRepo, packaging.Requirement, CompatHTTPClient |
| `InstallService` | git clone 节点 / uninstall | git portable, ScannedNodeRepo, VersionRepo |
| `CatalogHTTPClient` | 拉远程节点目录 + 缓存 | HTTPClient, CatalogRepo |
| `CompatHTTPClient` | 外部冲突知识查询（M3 留 hook） | HTTPClient |
| `HTTPClient` | urllib + 重试/超时/UA | stdlib only |

### 2.2 全局约束

- **架构模式沿用 M0/M1/M2**：Result[T] 返回值、EventBus 进程内事件总线、per-env service 实例（除 HTTP client 和 InstallService 外）
- **不引入新外部依赖**：HTTP 用 stdlib urllib；toml 解析用 stdlib tomllib（Python 3.11+）或 tomli
- **i18n**：所有用户可见字符串用 `qsTr()`，T23 抽取 .ts
- **trailing newline**：所有 .py / .qml 文件末尾保留
- **trailing comma + double quote**：Python 代码风格沿用 M2
- **theme color 不要 `||` fallback**：Theme.color(key) 已确认返回 truthy "#000000" for missing keys
- **JS 保留字**：用 `obj["class"]` 不用 `obj.class`
- **测试覆盖**：每个 service 至少 5 个测试，覆盖正常 + 关键错误路径
- **git 不依赖系统**：项目自带 portable 版本在 `bin/git-portable/`（.gitignore 排除）

## 3. 数据层（schema v4 增量）

### 3.1 `version_history` — 节点版本变更历史

| 字段 | 类型 | 约束 / 说明 |
|---|---|---|
| `id` | TEXT PK | 默认 `vh-{uuid.uuid4().hex[:8]}` |
| `env_id` | TEXT FK | → `environments.id` ON DELETE CASCADE |
| `package` | TEXT NOT NULL | 节点包名 |
| `action` | TEXT NOT NULL | enum: `upgrade` / `downgrade` / `lock` / `unlock` / `install` / `uninstall` |
| `version_before` | TEXT NULL | commit SHA / tag |
| `version_after` | TEXT NULL | commit SHA / tag |
| `result` | TEXT NOT NULL | enum: `success` / `failed` / `rolled_back` |
| `error_message` | TEXT NULL | 失败原因 |
| `performed_at` | TEXT NOT NULL | ISO timestamp |

**索引：** `idx_version_history_env(env_id)`、`idx_version_history_package(env_id, package)`、`idx_version_history_performed_at`

### 3.2 `dep_records` — 节点依赖快照

| 字段 | 类型 | 约束 / 说明 |
|---|---|---|
| `id` | TEXT PK | 默认 `dr-{uuid8}` |
| `env_id` | TEXT FK | → `environments.id` ON DELETE CASCADE |
| `package` | TEXT NOT NULL | 节点包名 |
| `source` | TEXT NOT NULL | enum: `requirements_txt` / `pyproject_toml` / `install_py` |
| `dep_name` | TEXT NOT NULL | 依赖包名（如 `torch`） |
| `dep_version_spec` | TEXT NULL | PEP 508 spec（如 `>=2.0,<3.0`） |
| `scanned_at` | TEXT NOT NULL | ISO timestamp |

**索引：** `idx_dep_records_env(env_id)`、`idx_dep_records_dep(env_id, dep_name)`

**唯一性：** `(env_id, package, source, dep_name)` 唯一（UNIQUE 约束），重新扫描覆盖

### 3.3 `catalog_cache` — 远程节点目录本地缓存

| 字段 | 类型 | 约束 / 说明 |
|---|---|---|
| `id` | TEXT PK | 默认 `cc-{uuid8}` |
| `source_url` | TEXT NOT NULL | 例如 `https://api.comfy.org/.../custom-node-list` |
| `package` | TEXT NOT NULL | 节点包名 |
| `raw_metadata` | TEXT NOT NULL | JSON（远程 API 原始 dict） |
| `cached_at` | TEXT NOT NULL | ISO timestamp |
| `expires_at` | TEXT NOT NULL | ISO timestamp（TTL 计算） |

**索引：** `idx_catalog_cache_source(source_url, package)`、`idx_catalog_cache_expires(expires_at)`

**唯一性：** `(source_url, package)` 唯一

### 3.4 Schema migration 策略

- 沿用 M0/M1/M2 模式：`CREATE TABLE IF NOT EXISTS` + `INSERT OR IGNORE INTO schema_version`
- v4 在 `CURRENT_SCHEMA_VERSION` 加 1
- 已存在的 `scanned_nodes`、`node_conflicts`、`node_meta_cache` 表**不动**
- v3 → v4 自动创建 3 个新表，旧数据 0 迁移
- 新增 3 个 repo：`VersionRepo`、`DepRepo`、`CatalogCacheRepo`

### 3.5 `scanned_nodes` 表增量列（M3 加 1 列）

| 字段 | 类型 | 说明 |
|---|---|---|
| `locked` | INTEGER NOT NULL DEFAULT 0 | 0 = 未锁定, 1 = 锁定（升级时跳过） |

v3 → v4 迁移时用 `ALTER TABLE scanned_nodes ADD COLUMN locked INTEGER NOT NULL DEFAULT 0`。

## 4. 服务层

### 4.1 `VersionService` — 节点版本管理（本地 git）

```python
class VersionService:
    def __init__(
        self, *,
        version_repo: VersionRepo,
        scanned_repo: ScannedNodeRepo,
        conn: sqlite3.Connection,
        event_bus: EventBus,
        git_exe_resolver: Callable[[], Path | None],
    ): ...

    def list_status(self, env_id: str, package: str) -> Result[list[VersionStatus]]:
        """返回 [{current_sha, has_update, locked, ...}]"""

    def upgrade(self, env_id: str, package: str, *, target: str | None = None) -> Result[VersionRecord]:
        """git fetch + reset --hard {target or origin/HEAD}"""

    def downgrade(self, env_id: str, package: str, target: str) -> Result[VersionRecord]:
        """git checkout {target}"""

    def lock(self, env_id: str, package: str) -> Result[ScannedNode]:
        """scanned_nodes.locked = 1"""

    def unlock(self, env_id: str, package: str) -> Result[ScannedNode]:

    def rollback(self, env_id: str, package: str, history_id: str) -> Result[VersionRecord]:
        """git checkout {history.version_before}"""

    def list_history(self, env_id: str, package: str, *, limit: int = 50) -> Result[list[VersionRecord]]:

    # EventBus: emit 'versionChanged(env_id, package)' on success/fail
```

**git_exe_resolver：** callable 返回 git 可执行文件路径，检测顺序：
1. `bin/git-portable/cmd/git.exe` （项目内 portable）
2. `shutil.which("git")` （系统 PATH）
3. 都找不到 → 抛 `GitNotFoundError`，bridge 转 `Result.fail(GIT_PORTABLE_MISSING)`

**git 操作：** `subprocess.run([git_exe, ...], cwd=pkg_dir, timeout=60, capture_output=True, text=True, encoding="utf-8")`
- 非零退出 → `Result.fail(GIT_FAILED)` + stderr 前 200 字
- 超时 → `Result.fail(GIT_TIMEOUT)`
- dirty working tree → `Result.fail(GIT_DIRTY)`（不自动 stash，提示用户）

### 4.2 `DepService` — 依赖解析（本地）

```python
class DepService:
    def __init__(
        self, *,
        dep_repo: DepRepo,
        scanned_repo: ScannedNodeRepo,
        conn: sqlite3.Connection,
        event_bus: EventBus,
        compat_client: CompatHTTPClient,
    ): ...

    def scan_deps(self, env_id: str, package: str) -> Result[list[DepRecord]]:
        """读 pkg_dir/requirements.txt + pyproject.toml [project.dependencies]
           + install.py（best-effort），写入 dep_records（覆盖该 package 旧记录）"""

    def list_deps(self, env_id: str, package: str | None = None) -> Result[list[DepRecord]]:

    def detect_conflicts(self, env_id: str) -> Result[list[Conflict]]:
        """本地冲突：同 env 内不同 package 对同一 dep_name 有不兼容的
           dep_version_spec（用 packaging.requirements.SpecifierSet 判定）"""

    def check_global(self, env_id: str) -> Result[list[Conflict]]:
        """把 dep_records 发给 CompatHTTPClient，查全局冲突知识
           （M3 base_url 为空时直接返回 []）"""

    # EventBus: emit 'depsChanged(env_id)'
```

**复用 M2 的 Conflict 模型 + ConflictService：** 新增 `conflict_type` 枚举值：
- `local_dep_version` — 同 env 内本地约束冲突
- `global_dep_known_incompat` — 外部知识库标记的不兼容

**解析策略：**
- `requirements.txt` — `pathlib.Path.read_text()` + 逐行正则（容忍注释和 `-r` include 指令简化跳过）
- `pyproject.toml` — `tomllib.loads(content)["project"]["dependencies"]`（Python 3.11+）或 `tomli.loads(...)`
- `install.py` — M3 暂不解析（标 `source=install_py` 但 `dep_records` 不写入，留 M4+）

### 4.3 `InstallService` — 节点安装 / 卸载

```python
class InstallService:
    def __init__(
        self, *,
        scanned_repo: ScannedNodeRepo,
        version_repo: VersionRepo,
        conn: sqlite3.Connection,
        event_bus: EventBus,
        git_exe_resolver: Callable[[], Path | None],
    ): ...

    def install_from_git(
        self, env_id: str, repo_url: str, *, target_dir_name: str | None = None,
    ) -> Result[ScannedNode]:
        """git clone {repo_url} {custom_nodes_path}/{target_dir_name or last path segment}
           + 写入 scanned_nodes + version_history(action=install)
           + 触发 ScannedNodeService.scan"""

    def install_from_catalog(
        self, env_id: str, catalog_entry: CatalogEntry,
    ) -> Result[ScannedNode]:
        """从 catalog_entry 取 repo_url，调 install_from_git"""

    def uninstall(self, env_id: str, package: str) -> Result[None]:
        """删除 custom_nodes_path/{package} + 写 version_history(action=uninstall)
           + 删 scanned_nodes 行"""

    # EventBus: emit 'nodeInstalled' / 'nodeUninstalled'
```

**目标目录命名：** 取 `repo_url` 最后一段（如 `https://github.com/foo/bar.git` → `bar`），去除 `.git` 后缀

**冲突处理：** 目录已存在 → `Result.fail(INSTALL_DIR_EXISTS)`，UI 弹窗让用户确认覆盖（覆盖需先 uninstall 再 install）

### 4.4 `CatalogHTTPClient` — 远程节点目录

```python
class CatalogHTTPClient:
    def __init__(

           否则 HTTP GET → 写 cache → 返回"""

    def search_remote(self, query: str, *, limit: int = 20) -> Result[list[CatalogEntry]]:
        """本地 cache 内 substring 匹配（大小写不敏感），不发起 HTTP"""

    def get_remote(self, package: str, *, force_refresh: bool = False) -> Result[CatalogEntry]:

    def get_cached(self, package: str) -> Result[CatalogEntry]:
        """读 cache，stale 也返回，带 stale=True 标记"""
```

**降级：** HTTP 失败 → 返回 cache + `stale=True`，UI 显示"离线（缓存于 {time}）"

### 4.5 `CompatHTTPClient` — 外部冲突知识库

```python
class CompatHTTPClient:
    def __init__(self, *, base_url: str, http_client: HTTPClient): ...

    def check_known_incompat(self, deps: list[DepSpec]) -> Result[list[KnownIncompat]]:
        """POST {base_url}/compat/check，body 是 dep 列表
           M3 阶段 base_url 为空时返回 Result.fail(API_NOT_CONFIGURED)"""
```

**M3 阶段 `compat_api_base_url` 在 `settings.json` 里留空字符串占位**，UI 不暴露"检查全局已知冲突"按钮（或按钮置灰）。`CompatHTTPClient.check_known_incompat()` 在 base_url 为空时返回 `Result.fail(API_NOT_CONFIGURED)`。M4+ 决定 API 提供方后再激活。

### 4.6 `HTTPClient` 基类（infra 层）

```python
class HTTPClient:
    def __init__(
        self, *,
        timeout: float = 10.0,
        max_retries: int = 3,
        user_agent: str = f"ComfyUI-Manager/{__version__} (+https://github.com/fogyisland/...)",
    ): ...

    def get(self, url: str, *, params: dict | None = None) -> Result[dict | bytes]:
        """GET，Content-Type=application/json 自动解析，其他返回 bytes"""

    def post(self, url: str, *, json_body: dict | None = None) -> Result[dict]:
        """POST application/json"""

    def head(self, url: str) -> Result[int]:
        """返回 status code（用于 ETag 检查等）"""
```

**重试策略（指数退避 1s/2s/4s）：**

| 错误类型 | 重试 |
|---|---|
| Timeout | ✅ |
| ConnectionError | ✅ |
| 5xx | ✅ |
| 429 Too Many Requests | ✅（等 Retry-After header） |
| 4xx (除 429) | ❌ |
| JSON 解析失败 | ❌ |

**实现：** `urllib.request` + 自定义 opener，**不引入 requests / httpx**

### 4.7 Git portable 准备

**存放：** `bin/git-portable/`（Git for Windows Portable 2.47.0，约 50MB）
- `bin/git-portable/cmd/git.exe`
- `bin/git-portable/VERSION` — 内容是 `2.47.0`（启动时校验）

**下载：** `scripts/fetch_git_portable.py`
- 调用 `urllib` 从 `https://github.com/git-for-windows/git/releases/download/v2.47.0.windows.1/PortableGit-2.47.0-64-bit.7z.exe` 下载
- 解压到 `bin/git-portable/`（用 stdlib `subprocess` 调 7z 或自解压 .exe）
- 校验 VERSION 文件

**.gitignore 增量：**
```
# M3 git portable
bin/
```

**启动检测（AppContext.__init__）：**
- 读 `bin/git-portable/VERSION`，存在则用 `bin/git-portable/cmd/git.exe`
- 否则 fallback 到 `shutil.which("git")`
- 都没有 → 不阻塞启动，但 VersionService / InstallService 调用时返回 `GIT_PORTABLE_MISSING`

## 7. 错误码表（M3 增量）

| 错误码 | 触发场景 | UI 表现 |
|---|---|---|
| `GIT_PORTABLE_MISSING` | 项目无 git portable 且系统无 git | 红条 "请运行 scripts/fetch_git_portable.py 下载 git" |
| `GIT_PORTABLE_CORRUPT` | VERSION 文件不匹配或关键 exe 缺失 | 红条 "git portable 损坏，请重新下载" |
| `GIT_TIMEOUT` | subprocess 超时（>60s） | 黄条 "升级超时，请检查网络" |
| `GIT_FAILED` | git 命令非零退出 | 红条 + stderr 前 200 字 |
| `GIT_DIRTY` | working tree 有未提交修改 | 红条 "检测到未提交修改，请先 commit 或 stash" |
| `GIT_NO_REMOTE` | 节点目录不是 git 仓库 | 红条 "该节点不是 git 来源，无法升级" |
| `DEP_PARSE_FAILED` | requirements.txt 或 pyproject 语法异常 | 黄条 "依赖文件解析失败：{file}: {detail}" |
| `DEP_NO_REQUIREMENTS` | 节点无依赖声明 | 静默（不显示） |
| `INSTALL_FAILED` | git clone 失败 | 红条 + stderr 摘要 |
| `INSTALL_DIR_EXISTS` | 目标目录已存在 | 弹窗 "目录已存在，是否先卸载再安装？" |
| `HTTP_TIMEOUT` | 10s 内无响应（重试 3 次后） | 黄条 "远程超时" |
| `HTTP_STATUS_4XX` | 4xx 响应 | 红条 "远程拒绝请求：{status}" |
| `HTTP_STATUS_5XX` | 5xx 响应（重试后） | 黄条 "远程服务异常" |
| `HTTP_CONNECTION_FAILED` | DNS / TCP 失败（重试后） | 黄条 "离线（缓存于 {time}）" |
| `CACHE_WRITE_FAILED` | catalog_cache 写失败 | 静默降级（远程数据仍可用） |
| `API_NOT_CONFIGURED` | compat_api_base_url 为空 | 按钮置灰 |

### 10.3 已知风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| git 升级引入 breaking change | 高 | 每次升级前记录 version_before；rollback 可一键回 |
| ComfyUI Manager 官方 API 变更 | 中 | 错误码 + UI 提示；不影响本地功能 |
| requirements.txt 解析不完整 | 中 | 用 `packaging.requirements.Requirement`（已是 poetry 依赖） |
| 用户手动改 custom_nodes/ 下的 .git | 低 | 错误码 GIT_FAILED + 提示 stash |
| 离线模式 UI 误导 | 低 | 顶部黄条明示 "缓存于 {time}" |
| git portable 体积 ~50MB | 中 | 排除 git，但首次 setup 联网下载 |
| git 版本陈旧 | 低 | VERSION 文件 pin 具体 release；M4+ 提供 in-app 更新 |
| fetch 脚本被防火墙拦 | 中 | 文档提示用户手动下载并解压到 bin/git-portable/ |
